# V1.2 澄清、Brief 与设计确认门

## 适用范围与目标

本规则适用于 `create`、`convert` 和重大 `repair`。其目标不是重复提问，而是让用户在完整生成前明确确认：系统理解的任务、模板与视觉风格，以及代表性样片。`check_only` 保持只读，不进入这些生成门；轻微 `repair` 仅在改变内容或视觉方向时按风险进入相应门。

## 状态机

`input_confirmed -> clarification_complete -> brief_presented -> brief_confirmed -> design_card_presented -> design_confirmed -> deck_plan_ready -> sample_confirmed | sample_explicitly_skipped -> full_generation_allowed`

- 每一状态必须在当前对话中有可引用的用户回答、或在生成日志中记录其确认依据；不得由模型推定。
- 用户要求修改 Brief、模板、风格、资产来源策略或内容边界后，受影响状态退回 `brief_presented` 或 `design_card_presented`，必须重新确认。
- “已完整输入”只能省略缺字段问题，不能跳过 `brief_presented -> brief_confirmed`。

## 门 1：最小澄清

依据 `rule-intake.md` 仅询问缺失字段。将问题合并为一轮，优先使用用户能快速回答的格式，例如：

```text
我已确认输入：<路径或当前请求内容>。
仍需确认：
1. 场景和主要受众：
2. 希望达成的目标/核心主张：
3. 输出格式、比例与页数范围：
4. 内容可否重排、改写或删减：
5. 图片资产：仅用户素材 / 可用确认的本地资产 / 可用图解图表 / 是否允许 AI 生成（不自动外部检索）：
```

答案足以消除影响产物的歧义时，标记 `clarification_complete`。不因用户未指定模板或风格而停在此门：下一门必须展示选择卡。

## 门 2：解析 Brief 确认

在使用确认输入形成页纲、样片或完整产物前，展示以下 Brief；缺省值必须显示为“待确认”，不得伪装为选择结果：

```markdown
## 本次演示稿 Brief（待确认）

- 模式与已确认输入：
- 场景、主要受众：
- 目标与核心主张：
- 输出格式、比例、页数范围：
- 内容处理边界：
- 固定页适用性：
- 图片资产来源策略与限制：
- 仍待设计选择：模板、视觉风格（或智能推荐）
```

只有用户明确回复“确认”、逐项确认或给出等价授权（例如“Brief 没问题，按此继续”）才进入 `brief_confirmed`。如用户改写任一字段，更新 Brief 后再次等待确认。

## 门 3：模板与视觉风格确认

在 Brief 被确认后，必须按 `rule-template-selection.md` 显示模板卡和 9 套视觉风格卡及预览资源。用户可：

1. 选择一个模板和一个风格；
2. 指定分页面/分章节风格映射；
3. 明确选择“智能推荐”。

“智能推荐”只允许系统基于 Brief 给出 `推荐模板 + 推荐风格 + 简短理由`，然后等待用户明确确认该组合。它不是默认选择，也不授予完整生成权限。未确认时不得用场景默认值替代用户选择。

## 门 4：样片确认与受限例外

在 `brief_confirmed` 与 `design_confirmed` 之后，先提供一页最能代表视觉风险、内容风险或核心表达的样片，并说明其表达类型、视觉载体和资产来源。样片渲染可用于确认视觉，不得被误称为完整交付。

用户明确确认样片后，标记 `sample_confirmed`。唯一允许的例外是用户明确提出跳过样片并给出原因，例如：

```text
Brief、模板和风格都已确认；我接受该页纲，因 <具体原因> 跳过样片，请直接生成完整 PPTX。
```

这时标记 `sample_explicitly_skipped`，在日志记录原话或等价证据、原因、时间和仍适用的验证门。以下都不构成跳过：未回复、仅说“快一点”、系统推荐、旧任务的同意、默认设置或只选择模板/风格。跳过样片绝不允许跳过 Brief 或模板/风格确认。

## 不可写门与允许的中间物

在 `brief_confirmed` 前，不得写入 `Design.md`/`Content.md`、生成日志、样片、完整 PPTX/HTML，除非用户已明确授权创建工作文件来记录其输入。

在 `design_confirmed` 前，不得写入样片或完整 PPTX/HTML，不得静默套用模板/风格。可展示 Brief、选择卡和只读页纲提案。

在 `sample_confirmed` 或 `sample_explicitly_skipped` 前，不得写入完整 PPTX/HTML。可在确认后的日志中记录 Brief、选择和样片决定。

这些限制不妨碍 `check_only` 的只读检查，也不授权读取任何未确认来源。

## 记录要求

`create`、`convert` 和重大 `repair` 的生成日志必须记录：缺字段与回答、Brief 内容及确认依据、模板/风格选择卡或智能推荐内容、用户确认、分页面映射、资产来源策略、样片路径和确认/跳过决定。没有这些记录时，交付校验不得把完整生成视为通过。
