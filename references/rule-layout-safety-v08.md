# v0.8 排版安全方案

## 目标

v0.8 的核心目标是把 PPT 生成从“生成后人工看一下”升级为“生成前有内容预算、生成时有自适应排版、生成后有硬校验”。所有 TGO/GTLC PPTX 或 HTML 输出都必须避免文本越界、遮挡、异常换行、字号过小、Logo/页脚碰撞和内容完全跑出框架。

## 约束总览

- 生成前：在页数大纲中记录每页的内容密度、标题行数、正文条数、版式风险和必要拆页策略。
- 生成时：所有标题、正文、注释、表格说明都必须经过统一的排版安全规则，不允许直接把长文本塞进固定框。
- 生成后：PPTX 必须运行静态版型检查脚本，并渲染或截图检查每一页；有 `FAIL` 的页面必须返工。
- 交付前：生成日志必须记录逐页检查结果、修复动作、生成子智能体、校验子智能体、`检查风格` 和 `检查文字` 结论。

## 子智能体分工

- 生成和校验必须职责分离。
- `生成内容` 负责样片和完整 PPTX/HTML 生成，不能批准自己的输出。
- `检查风格` 和 `检查文字` 是校验 subagent，必须与 `生成内容` 不同。
- `检查风格` 只检查视觉、模板、版式、安全区、渲染和 v0.8 排版安全结果。
- `检查文字` 只检查中文表达、逻辑、来源忠实度、标题准确性、内容密度和拆页必要性。
- 如果 subagent 工具不可用，必须在生成日志中记录限制；可以退化为主流程的两次分离自检，但不能声称已完成独立 subagent 校验。

## 默认内容预算

- 封面页主标题最多 2 行；副标题、讲者、日期不能与主标题互相遮挡。
- 章节页标题最多 2 行；辅助说明不超过 2 条。
- 常规内容页标题最多 2 行，正文建议不超过 5 条 bullet；每条 bullet 建议不超过 28 个中文字符或等价长度。
- 双栏内容页每栏建议不超过 4 条 bullet；左右栏高度必须视觉平衡。
- 表格页优先控制在 6 行 x 5 列以内；超过时必须拆页、摘要化或改为重点指标卡片。
- 时间线、流程图、组织结构页优先减少节点文字；节点内文案超过 2 行时必须改为详情页或注释区。
- 复盘总结、金句页可以使用大字号，但主视觉文字不得压住说明文字、Logo 或页脚。
- 封面页必须先确认模板的文字安全区和主视觉禁入区；副标题、日期、讲者、赞助身份等辅助文字不得侵入 GTLC 主视觉或背景识别层。
- 固定页也计入预算：第 2 页是空白 `嘉宾介绍` 页，末页是 `感谢聆听` 页；除非用户明确提供内容或改写要求，否则不得留下说明性占位文字。
- 深色封面/封底若使用大幅 GTLC 背景主视觉，优先采用“弱模糊背景 + 居中纯实底内容框”方案：背景可轻度模糊和压暗，内容框必须完全不透明，框内不得透出背景。
- 首尾页内容框必须按文本实测宽高自适应，保持上下留白均衡；不得因固定框尺寸导致错行、超框、裁切或异常换行。
- `左图 + 项目基本信息` 页型用于一张现场/项目佐证图配右侧事实列表：左图为主视觉，右侧为 `项目基本信息` 标题和 3-6 条短 key-value 信息项；每条信息项默认必须单行，不能因为文本框高度足够就允许折行。

## 字号与安全区

- 默认最小字号：正文 20pt，辅助说明 16pt，表格/脚注 14pt。
- 硬性下限：任何可读正文不得低于 12pt；低于 14pt 必须在生成日志中说明原因。
- 标题如果换行，必须重新计算下方元素位置；不能让第二行标题盖住正文。
- 16:9 默认保留页面外边距不小于 0.45 inch。
- 当画布为 `13.333 x 7.5 in` 半尺寸时，所有坐标、字号阈值、页脚/Logo/主视觉禁入区按 `0.5` 缩放；缩放后仍必须通过渲染检查。
- 顶部右侧 Logo 区域和底部页脚区域是文本禁入区；正文、标题、图表标签不得进入这些区域。
- 封面和章节页的背景主视觉也是文本禁入区；静态检查脚本无法识别的模板视觉区，必须在人工 `检查风格` 中逐页确认。
- 首尾页关键文字必须显式指定字体，不得依赖主题字体或渲染器字体回退；生成后必须用实际渲染图确认没有跨平台字体导致的错行。
- 项目基本信息、时间地点、合作身份、负责人、日期等短 key-value 信息项默认按单行处理；除非页数计划明确允许折行，否则必须通过扩大文本框、删去多余空格、压缩表达或合理降字号来避免折行。
- `左图 + 项目基本信息` 页型中，右侧事实列必须避开 Logo/页脚禁入区，行文本框应使用统一 x、统一或近似统一宽度，并按最长一行预留宽度。半尺寸 `13.333 in x 7.5 in` deck 中，事实行文本区域通常不小于 `3.7 in`；全尺寸按比例不小于 `7.4 in`。
- 背景图、装饰元素可以进入安全区，但不能影响文字可读性。

## 溢出处理优先级

当内容无法放入当前版式时，按以下顺序处理：

1. 缩短标题、压缩 bullet 或合并重复表达。
2. 在安全区内增大文本框或调整布局比例。
3. 适度降低字号，但不得突破硬性字号下限。
4. 改为两栏、卡片、表格、时间线、摘要页等更适合的结构。
5. 拆成多页，并在页标题中保持清晰的连续关系。
6. 如果用户要求保留全部原文且无法安全放入，必须说明风险并请求确认，不能带着溢出交付。

## 生成实现要求

- PPTX 生成代码应封装统一的文本放置函数，例如 `fit_text_to_box`、`estimate_text_lines`、`split_slide_if_overflow`。
- 所有文本 shape 创建前必须先估算行数和高度；创建后必须检查位置、尺寸、字号和碰撞。
- 每个版式必须声明可用文本区域；封面、章节、封底等视觉型页面还必须声明主视觉禁入区，文本放置函数只能在安全区内扩框、缩字或换版式。
- 纯实底内容框生成时必须先测量最长文本行，再决定框宽；框高由文字组高度、分隔线和上下 padding 共同决定，不能先固定框再硬塞文字。
- `左图 + 项目基本信息` 生成时必须先测量右侧最长事实行，再确定右栏宽度和每行文本框宽度；如果最长行无法单行容纳，应先压缩日期空格、拆分 `时间/地点` 等复合行，或扩大右栏，而不是让 PowerPoint 自动换行。
- 不允许用多个独立文本框模拟同一段正文，除非它们有明确布局关系并经过重叠检查。
- 表格、流程图、时间线等结构化内容必须按真实元素数量选择布局；不能只靠缩小字号硬塞。
- HTML 输出必须用固定 16:9 canvas、CSS 变量和响应式约束模拟同样的安全区。

## 校验流程

PPTX 输出必须执行：

1. 运行 `scripts/check_pptx_layout.py` 做静态检查。
2. 渲染或截图每一页，生成可快速浏览的 contact sheet。
3. 静态检查必须覆盖半尺寸阈值缩放、固定 `嘉宾介绍`/`感谢聆听` 页、Logo/页脚禁入区、封面主视觉禁入区和文本高度估算。
4. 静态检查必须保留短信息项折行提醒；`SHORT_INFO_UNEXPECTED_WRAP` 代表短 key-value 信息可能发生非预期折行，需要扩大文本框、精简文字或在日志里记录明确例外。
5. 静态检查必须识别 `项目基本信息` 页型；`PROJECT_INFO_*` 代表右侧事实列存在折行、文本框过窄、未处于右栏或未关闭自动换行等风险。
6. 对所有 `FAIL` 页面返工；对 `WARN` 页面人工判断并记录。
7. 再运行与 `生成内容` 不同的 `检查风格`：重点查模板、颜色、字体、Logo、页脚、安全区、主视觉禁入区、错位、遮挡、异常换行。
8. 再运行与 `生成内容` 不同的 `检查文字`：重点查逻辑、忠实度、标题准确性、密度、重复表达和是否需要拆页。

HTML 输出必须执行：

1. 用浏览器预览桌面、移动和打印状态。
2. 检查固定 16:9 画布、标题/正文层级、安全区、Logo/页脚位置和文本溢出。
3. 对移动预览中出现的遮挡、裁切或滚动异常进行修复。

## 生成日志要求

生成日志必须新增并记录：

- 使用的排版安全版本：`v0.8`。
- 每页内容预算：标题行数、正文条数、预计风险、是否拆页。
- 静态检查命令和结果。
- 每页 `PASS/WARN/FAIL` 状态。
- `生成内容`、`检查风格`、`检查文字` 的 subagent 分工，以及生成与校验是否为不同 subagent。
- 所有返工动作。
- 仍需用户确认的低字号、超密度或保真度问题。

## 验收标准

- 没有文本框越出画布。
- 没有标题、正文、图表标签互相遮挡。
- 没有文字进入 Logo 或页脚禁入区。
- 没有封面/章节文字进入模板主视觉禁入区。
- 首尾页纯实底内容框内不透背景，文本不换行、不超框，上下留白均衡。
- 没有不可读小字号。
- 没有因为标题换行导致下方内容被覆盖。
- 没有未计划的短信息项折行，例如项目基本信息里的时间、主题、参会人群、合作身份等。
- `左图 + 项目基本信息` 页型通过 `PROJECT_INFO_*` 检查：事实列在右栏、事实行宽度足够、自动换行关闭或有明确例外、渲染图中每行均保持单行。
- 没有内容完全跑出框架。
- 第 2 页为合规的 `嘉宾介绍` 页，末页为合规的 `感谢聆听` 页，除非用户确认并记录了例外。
- 有 `FAIL` 的 PPTX/HTML 不得交付。
