# Style 9: Enterprise AI Consulting

Load when the user chooses visual style `9`, or when the topic is AI native organization, enterprise AI transformation, AI operating system, workflow automation, agent collaboration, or organization-level AI adoption.

Source basis: `TGO_AI_Design_System_v1.0.md`.

## Use For

AI native organization sharing, enterprise AI strategy, CTO/CEO conference talks, AI operating-system explanations, workflow/agent architecture, and organization transformation frameworks.

## Keywords

Enterprise AI, minimal technology, consulting framework, Apple Keynote, architecture diagram, system thinking, professional CTO conference.

## Visual Direction

- Prefer white or very light backgrounds with strong black typography.
- Use AI blue `#1456D9` for systems, capability layers, arrows, and architecture blocks.
- Use green only for execution/success flow and orange only for human decision or key emphasis.
- Build slides around one core message, one framework, or one formula.
- Prefer cards, layers, arrows, pipelines, architecture blocks, and two-column comparisons.
- Use large whitespace, thin borders, minimal icons, and clear hierarchy.
- Avoid decorative graphics, marketing poster style, excessive gradients, neon entertainment effects, and dense paragraphs.

## Page Patterns

- **Formula page**: `Concept = Factor + Factor + Factor`; keep the formula centered and highlight the bottom conclusion.
- **Architecture page**: Input -> Information Layer -> AI Operation System -> Workflow Container -> Output -> Feedback Loop.
- **Process page**: 01 -> 02 -> 03 -> 04; each step has a short title, explanation, and output.
- **Comparison page**: Old Mode vs AI Native Mode; keep both columns visually balanced and end with one clear conclusion.
- **Summary page**: Problem -> Insight -> Framework -> Conclusion; keep the conclusion short and memorable.

## Content Rules

- One slide communicates one core idea.
- Use 3-5 supporting points and one conclusion per slide.
- Replace paragraphs with cards, labels, or framework nodes.
- Use short sentence titles and bold only the real keyword.
- When content becomes dense, split into a framework overview plus detail pages.

## TGO/GTLC Handling

- Prefer `white` or `light` templates for most pages; use `dark` only for AI system, platform architecture, or chapter emphasis pages.
- Preserve top-right logo, footer, and TGO/GTLC brand chrome.
- Keep the TGO/GTLC identity layer as the frame, then apply the Enterprise AI Consulting visual language inside the content area.
- Keep diagrams editable whenever possible.
