# Style 5: Data Intelligence

Load when the user chooses visual style `5`.

## Use For

Benchmarks, AI Coding evaluation, business analysis, operations analysis, and growth analysis.

## Keywords

Data, analysis, insight, growth, efficiency, metrics.

## Visual Direction

- Charts occupy at least 60% of data slides.
- Dashboard logic, rigorous hierarchy, clear metric grouping.
- Reference mood: Palantir, Snowflake, Databricks.

## TGO/GTLC Handling

- Prefer `white` or `light` templates for dense data.
- Keep chart geometry editable and legible.
- Use fewer metrics per slide when the page starts to wrap or crowd.

## Content Expression

### Image Treatment

- Use only confirmed user photos, licensed/approved images, or approved AI-generated visuals. Use contextual operational or product images sparingly, only to establish the source of a metric or a real-world outcome.
- Keep crops factual and neutral; let the chart remain visually dominant instead of adding decorative image panels.

### Diagram Language

- Use measurable funnels, metric trees, cohorts, pipelines, and cause-to-effect flows with explicit labels and editable geometry.

### Data Expression

- Use large, legible editable charts and dashboard groupings; state metric definitions, periods, units, baselines, and one analytical takeaway where relevant.

### Avoid

- Avoid chart junk, 3D charts, unsupported causal arrows, image-led slides without evidence, and decorative dashboards with unreadable metrics.
