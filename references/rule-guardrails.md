# V1.1 守护规则

- 不得改写、删除、弱化或移动 `SKILL.md` 的“重要说明和介绍”。
- 不得修改 `assets/**`、`references/design/**` 或 V1 排版算法，除非任务明确授权并增加结果级验证。
- `check_only` 必须保持源文件只读；修复/转换默认创建新文件，不静默覆盖。
- 覆盖、批量修改或外部写入必须遵循：预览目标 → 明确确认 → 执行 → 事后核对。
- `create`、`convert` 和重大 `repair` 不得跳过生成日志；普通 `check_only` 不得为了形式强制创建日志。
- 不得在用户未确认时替换 GTLC Logo。
- 不得交付存在已知文本溢出、裁切、重叠、不可读小字、Logo/页脚碰撞或未批准检查器 `FAIL` 的产物。
- 不得为了保留页数突破 V1 字号下限；内容过密按缩写、扩框、合规缩字、换版式、拆页、询问的顺序处理。
- 新建/转换 PPTX 不得跳过适用的 `嘉宾介绍` 和 `感谢聆听` 固定页；存在目录时保持标题页、嘉宾介绍、目录、正文顺序。
- 不得一次性加载全部设计引用文件。
- 不得为每个 rule 强制创建子智能体；也不得把同一上下文自检声称为独立验证。
- 不得把结构 fixture 声称为真实路由行为，不得把静态检查声称为视觉质量验证。
- 生产失败样本不得写入封存 holdout；只能进入后续回归集版本。
- 可预期 CLI 失败必须使用稳定错误码，不以裸 traceback 作为用户错误信息。
- 同一确定性操作最多重试两次；未改变失败条件不得盲重试。
- 字体缺失依次回退：思源黑体、苹方、微软雅黑、Helvetica Neue、Helvetica、Arial、sans-serif。
- 标准半尺寸 PPT 按 `0.5` 缩放坐标和阈值，并重新渲染确认。
