# 来源转换工作流

本文件在用户提供 PPT、PDF、Markdown、HTML 或粘贴内容时加载。先按 `references/workflow.md` 完成总流程，再按来源类型执行本文件对应段落。

## 通用步骤

1. 识别来源类型、内容结构和可复用资产。
2. PPT/PDF 来源必须先确认：`1. 修改内容并套模板` / `2. 只套模板不改内容`。
3. 如果选择只套模板，不改写文案、不重排叙事；只做必要排版安全修复。
4. 如果选择修改内容，先提出优化方向，再进入页数大纲。
5. 所有输出计划都必须包含 PPT 固定 `嘉宾介绍` 和 `感谢聆听` 页。
6. 生成前创建生成日志，生成后运行检查器和渲染复核。

## PPT 到 PPTX

- 使用选定模板 PPTX 作为底稿。
- 保留源页意图，不保留无关装饰。
- 源封面映射为封面页；议程映射为目录页；章节页映射为章节页；普通页映射为内容页、双栏页、图片页或信息图页。
- 原图和图表只有在 GTLC/TGO 安全区域中仍可读时才复用。
- 如果源文件已有强品牌约束，先询问是融合品牌还是完全改为 GTLC/TGO。
- 校验时比对源页是否都有明确映射、合并或删除记录。

## PDF 到 PPTX

- 先确认是否修改内容。
- 只套模板时，保留 PDF 文本顺序和页面含义；无法可靠抽取时使用保守图片或文本块。
- 修改内容时，先抽取结构并提出改写方案，等待用户确认。
- 渲染源 PDF 和生成 PPTX，对比是否漏字、裁图、丢表或产生异常折行。

## Markdown 到 PPTX

| Markdown 结构 | PPT 映射 |
| --- | --- |
| 第一个 `#` | 封面标题 |
| 顶部元信息 | 封面副标题、讲者、日期 |
| `##` | 新章节或新页标题 |
| `###` | 当前页内小标题 |
| 列表 | 正文要点 |
| 图片 | 图片区或佐证图 |
| 表格 | 摘要、拆页或表格页 |
| 代码块 | 仅在必要时使用高对比等宽块 |

密集 Markdown 按观点拆页，不按标题机械拆页。优先一页一个清晰信息。

## HTML 到 PPTX

- 先解析语义结构：标题、章节、文章、列表、表格、图片、图注。
- 默认忽略装饰性 CSS；用户要求高保真时再保留视觉细节。
- 表格和图表不可读时，摘要化或拆页。
- 不要默认把整页 HTML 截图塞进 PPT，这会牺牲可编辑性。
- 如果 HTML 本身是 slide deck，保留 slide 边界并套用 GTLC/TGO 风格。

## PPT/PDF/Markdown/HTML 到 HTML Slides

HTML slide deck 必须使用：

- 固定 16:9 slide surface。
- GTLC/TGO 设计变量。
- 页面类：封面、章节、介绍、目录、内容、空白、结束。
- 与选定模板一致的 Logo、页脚和背景位置。
- 打印样式尽量保持一页一个 slide。

网页浏览时可以等比缩放 slide 画布，不要把内部内容重排成普通文章，除非用户明确要网页而不是 HTML 演示稿。
