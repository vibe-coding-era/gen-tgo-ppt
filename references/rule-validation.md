# 交付校验规则

## PPTX 校验

1. 运行 `scripts/check_pptx_layout.py`。
2. 修复所有 `FAIL`。
3. 对所有 `WARN` 做人工判断，并写入生成日志。
4. 渲染或截图每一页，优先生成 contact sheet。
5. 逐页检查：
   - 标题、正文、图表标签没有重叠
   - 没有文本框越出画布
   - 没有异常换行、裁切、超框或不可读小字
   - 文本没有进入 Logo 和页脚禁入区
   - 封面/章节文字没有压住主视觉
   - 固定 `嘉宾介绍` 和 `感谢聆听` 页合规
   - 首尾页实底框不透背景
   - 项目基本信息页事实项不折行

## HTML 校验

1. 使用固定 16:9 slide canvas。
2. 桌面、移动和打印预览都要检查。
3. 文本不得溢出、裁切、覆盖后续内容或低于可读字号。
4. Logo、页脚、背景和模板层级应与选定设计规则一致。

## 交付回复

最终回复包含：

- 文件路径
- 预览路径
- 生成日志路径
- SSOT 路径或 SSOT 区块摘要
- 检查命令和结果
- 未解决事项

如果无法运行某项检查，必须说明原因。

## SSOT 交付检查

- 每个已加载 rule 都必须有规则子智能体交接摘要。
- 最终文件路径、预览路径、检查结果和遗留问题必须来自 SSOT 的最新记录。
- 如果 SSOT 中仍有未裁决冲突，不得交付。
- 如果缺少独立规则子智能体，必须说明工具限制和替代自检方式。
