# V1.1 交付校验规则

## 证据层级

- `structural_fixture`：schema、路径、静态规则或自洽 fixture，只证明结构门禁。
- `observed_model_behavior`：隔离上下文中的真实路由观测，只证明被测提示下的行为。
- `observed_output_quality`：真实产物、渲染和检查结果，才支持输出质量结论。

三者不得互相冒充。缺失、运行错误或零分母必须标 `INSUFFICIENT`，不能按通过处理。

V1.1 Harness 的 deterministic fixture 需要隔离的可写临时目录。可用 `--temp-root` 或 `GEN_TGO_PPT_TEMP_ROOT` 指向已存在且可写的目录；若探测失败，无写入门禁继续执行，依赖临时写入的门禁必须标 `INSUFFICIENT` 并返回退出码 `3`，不得降格为 PASS。

## 模式门禁

所有会读取内容或写产物的模式都必须先有明确执行意图；所有被使用的输入都必须记录归属。目录存在性、历史日志或记忆不能替代本次确认。

### `check_only`

- 源文件 hash/mtime 或其他证据表明未修改。
- 运行适用静态检查并逐页渲染/预览；无法渲染则标记降级。
- 报告范围、证据、PASS/WARN/FAIL、限制和建议。

### `repair`

- 保留来源或明确记录覆盖确认和回滚点。
- 对修改点做针对性回归，并对完整产物执行最终检查和逐页复核。
- 重大修复需更新生成日志；正式/高风险任务需独立复核。

### `convert` / `create`

- 已记录澄清、已确认输入及归属、页纲、样片决定、来源忠实度和日志。
- PPTX 运行 `scripts/check_pptx_layout.py`；HTML 运行 `scripts/check_html_layout.py`。
- 修复全部未批准 `FAIL`，人工判断 WARN，渲染或预览每一页。
- PPTX 固定页、目录顺序、实底面板、Logo/页脚安全区、字号和文本密度满足适用规则。

### `handoff`

- 未产生伪造产物或声称执行越界操作。
- 移交记录包含越界原因、已知输入、未执行操作和建议下一步。

## PPTX 逐页重点

检查文本越界/重叠/裁切、异常换行、不可读小字、Logo/页脚碰撞、标题页与封底实底框、嘉宾介绍/目录/正文顺序、项目基本信息短事实项折行。对用户批准的例外给出日志证据。

## HTML 重点

使用固定 16:9 slide canvas；静态检查后检查桌面、移动和打印预览。确认文本、Logo、页脚、背景、层级与所选设计一致。

## 回复合同

交付回复列出文件、预览、日志/SSOT（若适用）、命令、结果、证据层级和未解决事项。无法运行某项检查时说明原因与影响。
