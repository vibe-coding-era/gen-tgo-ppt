# 设计澄清与工作文件规则

## 启动澄清

- 首次使用当前目录，或缺少 `Design.md` 时，询问应用场景和规格。
- 首次使用当前目录，或缺少 `Content.md` 时，询问主题、问题和思考模式。
- 如果用户已经提供了明确答案，不重复询问。
- 如果源文件能直接看出页数、标题或嵌入图片，不把这些作为澄清问题。

## Design.md

在用户当前工作目录创建或更新 `Design.md`：

```markdown

```

## 数字菜单

所有菜单接受纯数字确认：

- 场景：`1. GTLC 大会` / `2. TGO日常活动分享`
- PPT/PDF 处理：`1. 修改内容并套模板` / `2. 只套模板不改内容`
- 输出格式：`1. PPT` / `2. HTML`

如果用户输入 `1`，映射到最近一次菜单的第一个选项。

## 少问原则

优先询问这些会改变结果的事项：

- 输出 PPTX、HTML，还是两者
- 内容保真还是允许重排优化
- 模板倾向：白底、浅色、深色、混用或自动选择
- 是否替换 Logo
- 是否替换会议年份、城市、标题、讲者、日期
- 是否允许拆页、压缩、改写标题
