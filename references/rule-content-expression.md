# V1.2 页级内容表达与视觉资产规则

## 目标与边界

本规则让内容表达先于版式填充：核心内容页先说明要传达的观点，再选择最适合的视觉载体。它要求更多有信息作用的图片与视觉表达，但不要求为每页硬塞装饰图；所有输出仍必须遵守 V1 排版安全、来源边界和用户确认的模板/风格。

本规则不接入外部图片、图库、搜索或下载服务。图片和视觉资产只能来自用户明确批准的来源策略；外部素材服务必须由用户另行明确授权后才可启用。

## 页级表达合同

页纲中每一张核心内容页至少记录以下字段：

| 字段 | 要求 |
| --- | --- |
| 表达类型 | `主张`、`数据`、`案例`、`流程`、`架构`、`对比`、`时间线`、`人物故事`、`决策` 或经说明的其他类型 |
| 核心信息 | 一句可被受众复述的结论、问题或行动 |
| 视觉载体 | `主视觉图片`、`信息图解`、`图表`、`数据表`、`结构框架`、`流程图`、`对比卡`、`时间线`、`截图` 或 `纯文字` |
| 资产来源 | `用户提供`、`用户确认的本地资产`、`AI 生成（已授权）`、`可编辑图解/图表` 或 `无视觉资产（说明原因）` |
| 风格约束 | 模板键、视觉风格编号及其适用的颜色、留白、图解/图片处理规则 |
| 文本预算 | 与 V1 安全规则相容的标题、正文和说明预算；密集内容须拆页或改用结构化表达 |

封面、章节页和核心叙事/案例页优先选择强主视觉；数据页优先选择可读图表；流程、架构和决策页优先选择可编辑图解；确实需要精确比较时才使用表格。纯文字只适用于短结论、引用或没有可验证视觉资产的内容，并在页纲说明原因。

## 视觉优先级

对于有三个或以上核心内容页的演示稿，规划时应使多数核心内容页具有信息型视觉载体（图片、图解、图表、结构框架、流程、对比或截图），并在不能做到时记录原因，例如用户要求逐字保真、没有可用资产、保密限制或事实表格不可替代。视觉表达优先级如下：

1. 能直接证明或解释核心信息的用户素材、截图、照片或图表；
2. 遵循所选风格的可编辑图解、结构框架、流程图或比较视觉；
3. 获用户授权的 AI 生成图片；
4. 纯文字或表格，并说明为何前述载体不适用。

不得用无关的通用配图代替证据、案例或逻辑。不得把装饰图片记为“信息型视觉”。

## 资产来源策略

在 Brief 中确认可用组合并写入日志；可选项包括：

- `仅用户素材`：只用当前请求中提供或用户明确点名的图片、截图、Logo 与文件。
- `用户素材 + 确认的本地资产`：可用用户明确授权的本地目录/模板资产，记录路径或资产 ID。
- `可编辑图解/图表`：可创建不依赖外部素材的形状、图表和结构图。
- `AI 生成（已授权）`：仅在当前宿主有相应能力且用户明确授权时使用；记录提示摘要、生成时间和用途。
- `不使用图片`：用户明确限制时，仍可按风格使用图表、图解和排版表达。

来源不明、版权状态不明或未在策略中批准的资产不得进入产物。用户提供的素材不自动意味着允许裁剪、重绘、生成衍生图或用于封面；此类限制应随资产记录。

## 风格一致性

按用户确认的视觉风格文件执行；模板只定义品牌框架和版式基底，风格决定内容区的图片处理、图解语法、数据表达和节奏。混用风格时必须在页纲中标注页面范围与转换理由。

最低要求如下：

- Style 1：克制留白、决策图表和结论卡，避免装饰性图片。
- Style 2：大标题、单一主张和高节奏主视觉，避免密集说明。
- Style 3：沉浸式全幅视觉与极少文字，必要的细节转入支持页。
- Style 4：用网络、流程或层级图解释 AI/Agent，不把渐变或粒子当作内容。
- Style 5：图表应占数据页主要面积，确保指标、单位和比较对象可读。
- Style 6：极简、编辑感与短句；图片稀少但有质感，避免堆叠元素。
- Style 7：克制的东方科技点缀，图片与图解服务产业/文化叙事，避免仪式化装饰。
- Style 8：高对比插画/漫画/创意图形可承担故事节奏，但正式内容页保持信息清楚。
- Style 9：用可编辑架构、层级、箭头、公式与对比框架解释企业 AI，避免营销海报式装饰。

风格文件与用户选择冲突时，以用户明确选择为准，但记录偏离项并在样片中先验证。

## 日志与校验

生成日志的页纲与样片段必须包含页级表达合同。交付前，逐页检查：视觉载体是否支持核心信息、资产来源是否符合策略、图片/图解是否遵循选择的风格，以及文本是否仍满足 V1 安全规则。来源未记录、风格不一致或把装饰图充当证据均为未批准问题。
