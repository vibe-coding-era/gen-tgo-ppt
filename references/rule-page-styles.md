# 可复用页型规则

## 首尾页纯实底内容框

适用：深色 GTLC 标题页、封底；章节页在背景主视觉会干扰文字识别时也适用。

- 背景：使用完整 GTLC 背景图，轻微模糊和轻微压暗，保持比例。
- 内容框：居中纯实底深蓝，推荐 `#061C4C` 或接近 GTLC 深蓝。
- 内容框必须完全不透明，框内不得透出背景图。
- 标题页和封底默认必须同时使用本页型，不能只套封底。
- 保留原生左栏标题页必须有用户明确确认，并写入页数计划、SSOT 和生成日志。
- 框宽按最长文本行加水平 padding 计算。
- 框高按文字组高度、分隔线和上下 padding 计算。
- 上下留白必须视觉均衡。
- 关键文字显式设置字体，不依赖主题字体。
- 如果文字不适配，优先扩大框或调整字号，不允许静默折行或超框。

## 左图 + 项目基本信息

适用：项目背景、活动现场、赞助结案、证据照片配事实说明页。

- 左侧放主照片或佐证图，作为页面主视觉。
- 右侧放 `项目基本信息` 标题和 3-6 条事实项。
- 事实项使用小圆点或等价标记 + 单一文本框。
- 常见标签：`大会主题`、`时间`、`地点`、`参会人群`、`合作身份`、`项目名称`、`负责人`、`主办方`。
- 每条事实项默认必须单行；不要因为文本框高度能装两行就允许折行。
- 放置前先测量最长事实行，并以最长行为依据设置统一或近似统一宽度。
- 半尺寸 `13.333 in x 7.5 in` 中，事实行文本区域通常不小于 `3.7 in`；全尺寸不小于 `7.4 in`。
- 日期和短语应压缩不必要空格，例如 `2026年6月27-28日｜地点：杭州`。
- 如果仍不能单行，优先扩大右栏、压缩表达、拆分时间/地点，再考虑合理降字号。
- 检查器中的 `PROJECT_INFO_*` 和 `SHORT_INFO_UNEXPECTED_WRAP` 都是需要处理的风险。

## 短 key-value 信息行

- `时间：...`、`地点：...`、`合作身份：...`、`负责人：...` 默认按单行处理。
- 如果计划允许折行，必须在页数计划和生成日志中说明。
- 不要只看垂直高度；两行短事实项即使不溢出，也会破坏页型节奏。
