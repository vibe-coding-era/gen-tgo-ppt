# V1.1 主工作流

`SKILL.md` 负责入口路由，本文件负责模式化执行。先读 `rule-routing-and-modes.md`，再按模式只加载必要规则。

## 阶段 0：启动、模式与边界

1. 说启动语。
2. 选定 `check_only`、`repair`、`convert`、`create` 或 `handoff`，记录输入、目标和禁止事项。
3. 识别文件类型；不重复询问源文件可直接给出的页数、标题、图片和比例。
4. `handoff` 直接形成移交记录并停止演示稿生成。

## `check_only` 路径

1. 保持源文件只读，定义检查范围和证据形式。
2. 按需读取模板、结构、排版与校验规则，不创建无关 `Design.md` / `Content.md`。
3. 运行适用检查器并渲染/预览；无法渲染时标记降级。
4. 输出逐页或逐问题的 PASS/WARN/FAIL、证据、限制和建议，不执行修复。

## `repair` / `convert` / `create` 路径

### 1. 最小澄清

- 按 `rule-intake.md` 只补不可从材料推断且会改变产物的问题。
- PPT/PDF 转换确认“修改内容并套模板”或“只套模板不改内容”。
- Logo 替换、源文件覆盖、样片跳过均需明确决定。

### 2. 内容与渐进加载

1. 读取任务所需的 `Design.md`、`Content.md` 和来源。
2. 先读设计索引与共享规则，再只读一个场景、相关模板和所选视觉风格。
3. 给出页数、逐页目的、布局、资产、文本预算和拆页策略。

### 3. 日志、样片与确认

1. `create`、`convert` 和重大 `repair` 在写产物前创建 V1.1 生成日志。
2. 生成一页代表性或最高风险样片并渲染检查。
3. 等待用户确认；明确跳过时在日志记录原因。

### 4. 完整生成或修复

1. 默认不覆盖来源，先写版本化新文件。
2. 新建或转换的 PPTX 应遵守固定页与结构规则。
3. 文本先做预算，再进版；特殊页型按 `rule-page-styles.md`。
4. 正式或高风险任务按 `rule-subagents.md` 分配独立通道；其余任务使用单 Agent 的分离步骤。

### 5. 验证与返工

1. PPTX 运行 `check_pptx_layout.py`；HTML 运行 `check_html_layout.py`。
2. 渲染/预览每页，处理全部未批准 `FAIL`；对 WARN 给出判断。
3. 失败时按 `rule-failure-recovery.md` 分类，只修最小责任层并重跑针对性检查。
4. 最后执行完整交付门禁；正式或高风险任务合并 SSOT。

## 交付

最终回复给出：模式、产物或被检查文件、预览证据、日志/SSOT（若适用）、检查命令与结论、降级或遗留问题。不得让结构门禁、模型行为与结果质量三种证据互相替代。
