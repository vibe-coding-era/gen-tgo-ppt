# V8.02 主工作流

本文件是执行 TGO/GTLC 演示稿任务的主流程。`SKILL.md` 只负责路由，具体规则按阶段加载。

## 阶段 0：启动与任务识别

1. 先说启动语：`我是 TGO鲲鹏会 PPT 生成 Skill，可以生成 PPT 格式和 HTML 格式。`
2. 判断任务类型：新建、套模板、修改内容、修复 PPT、检查 PPT、生成 HTML、或混合任务。
3. 如果用户给了 PPT/PDF/Markdown/HTML/图片/Logo，先识别文件类型与用途。
4. 如果用户只要求修复或检查既有 PPT，不重复询问已经从文件中可见的信息。

## 阶段 1：最小澄清

按 `references/rule-intake.md` 执行。只问会影响后续不可逆选择的问题：

- 场景：`1. GTLC 大会` / `2. TGO日常活动分享`
- 规格：默认 `16:9`，或用户指定比例
- 输出：`1. PPT` / `2. HTML`
- PPT/PDF 来源处理：`1. 修改内容并套模板` / `2. 只套模板不改内容`
- Logo：如果用户上传 Logo，确认是否替换 GTLC Logo

## 阶段 2：内容理解

1. 读取 `Design.md`、`Content.md` 和源文件。
2. 总结核心信息、受众、页数密度、结构弱点和版型风险。
3. 问用户是否需要先优化叙事结构、标题表达、页数密度或删减合并。
4. 如果用户选择只套模板，不主动改写内容；只做必要的拆分、适配和排版安全修复。

## 阶段 3：渐进式加载设计规则

1. 读取 `references/design/index.md` 和 `references/design/shared/design.md`。
2. 根据场景读取一个场景文件。
3. 根据模板读取一个或少量模板文件。
4. 根据用户选择读取相关视觉风格文件。
5. 不要一次性读取所有模板、场景和视觉风格文件。

## 阶段 3.5：规则子智能体与 SSOT

1. 读取 `references/rule-subagents.md` 和 `references/rule-ssot-handoff.md`。
2. 为本次已加载的每个 rule 文件分配不同规则子智能体。
3. 每个规则子智能体独立执行自己的规则，不代替其他规则判断。
4. 每个规则子智能体按统一交接格式输出 `PASS/WARN/FAIL`、风险、建议和证据。
5. `SSOT-合并` 汇总所有规则交接，形成唯一执行计划、冲突裁决和交付状态。
6. 如果工具策略不支持创建多个子智能体，必须在 SSOT 和生成日志中记录限制，并以分离自检替代。

## 阶段 4：页数大纲与预算

在生成样片前给用户确认：

- 总页数
- 每页标题、目的、布局、核心内容、资产
- 模板和视觉风格
- 标题/正文预计行数
- 溢出风险和拆页策略
- PPT 输出必须包含标题页后的 `嘉宾介绍` 页和最终 `感谢聆听` 页
- 默认开场顺序必须是第 1 页标题页、第 2 页 `嘉宾介绍`、第 3 页目录页、第 4 页起正文；不设目录页必须记录用户确认或短材料例外
- 目录页不得承载正文标题或核心论点句

## 阶段 5：日志与样片

1. 先按 `references/generation-log.md` 创建生成日志。
2. 生成一页代表性样片，优先选择最典型内容页或风险最高页面。
3. 渲染样片，检查标题、正文、图片、Logo、页脚和异常折行。
4. 等待用户确认样片风格、密度和内容处理。

## 阶段 6：完整生成

1. 使用选定 PPTX 模板或 HTML slide 结构生成完整文件。
2. 若可用，将生成交给 `生成内容` 子智能体。
3. 所有文本必须经过排版安全估算，不得把长文本直接塞进固定框。
4. 特殊页型按 `references/rule-page-styles.md` 执行。

## 阶段 7：检查与返工

1. PPTX 运行 `scripts/check_pptx_layout.py`。
2. 渲染或预览每一页，生成截图或 contact sheet。
3. 对所有 `FAIL` 返工；对 `WARN` 人工判断并记录。
4. 按 `references/rule-validation.md` 做交付检查。
5. 已加载的每个 rule 由对应规则子智能体重新提交交接结论。
6. 若可用，分别运行 `检查风格` 和 `检查文字`。
7. `SSOT-合并` 只采纳各规则交接后的唯一结论，最终回复不得绕开 SSOT。

## 阶段 8：交付

最终回复包含：

- 输出文件路径
- 预览或渲染结果路径
- 生成日志路径
- 检查命令和结果摘要
- SSOT 路径或 SSOT 区块摘要
- 未完成事项或明确说明“无”
