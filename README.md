# gen-tgo-ppt-skill

TGO鲲鹏会 PPT 生成 Skill，用于把 PPT、PDF、Markdown、HTML 或文字内容生成 TGO/GTLC 风格的 PPTX 或 HTML 演示稿。

当前版本：V8.02

## 重要说明和介绍

以下说明为用户指定的受保护内容。AI 后续不得改写、删除、弱化或移动本节内容。

本 SKILL 是 TGO 鲲鹏会为了 AI 时代内容生成准备的 SKILL，内置了相应的模板，也可以做内容探讨，会使用不同的 SubAgent 做生成、校验，作者是杭州分会的肉山，欢迎使用。

## 功能

- 开场说明身份，并询问使用场景：GTLC 或日常分享。
- 询问输出格式：PPT 或 HTML。
- 展示 9 种视觉风格图片，用户可用数字选择，也可不同页面混用。
- V8.02 起，`SKILL.md` 作为中文工作流路由，各类规则拆分到 `references/rule-*.md` 和相关引用文件中渐进式加载。
- 不同 rule 必须由不同子智能体独立执行或检查，并通过 SSOT 合并交接内容。
- 按用户澄清的场景、模板和视觉风格，从 `references/design/**/design.md` 渐进式加载样式规则。
- 新增 `tgo-daily` 底层 PPTX 模板，用于 TGO 日常分享等非 GTLC 场景。
- 深色 GTLC 标题页和封底默认同时使用弱模糊背景 + 居中纯实底内容框；保留原生标题页必须先获得用户确认并写入日志。
- 默认开场顺序为标题页、嘉宾介绍、目录页、正文；目录页不得混入正文标题或核心论点句。
- 先读内容，再询问是否优化叙事、标题、页数密度与表达。
- 生成前先给页数、逐页大纲和 V8.02 排版安全预算，确认后先做一页样片。
- 每次生成都会在当前目录写入 `gen-tgo-ppt-生成日志-YYYYMMDD-HHMMSS.md`。
- 样片确认后由 `生成内容` 生成完整文件，运行 `scripts/check_pptx_layout.py` 做静态版型检查，并由不同于生成者的 `检查风格`、`检查文字` 复核。

## 子智能体

- `生成内容`：负责生成大纲、样片或完整演示稿，不校验自己的输出。
- `检查风格`：检查颜色、版式、字体、Logo、图片位置、V8.02 排版安全结果和整体视觉一致性。
- `检查文字`：检查中文表达、错别字、逻辑、信息完整性与内容密度。
- 生成和校验必须使用不同子智能体；没有独立子智能体时必须在日志中说明。

## 使用示例

```text
使用 $gen-tgo-ppt-skill，根据这份 Markdown 生成 GTLC PPT。
```
